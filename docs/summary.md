# 项目总结

**持有型ABS申报项目动态跟踪爬虫 v2**

---

## 项目目标

自动爬取上交所和深交所的持有型ABS申报项目，与历史基准文件比对，标注新增和状态变更，导出 Excel 供人工审阅。

---

## 输出文件清单

| 文件路径 | 说明 |
|---|---|
| `data/output/持有型ABS项目汇总_YYYYMMDD_HHMMSS.xlsx` | 主输出，含3个Sheet |
| Sheet: 全部项目 | 基准+爬取合并后的完整列表 |
| Sheet: 新增项目 | 本次爬取到、基准中不存在的项目（绿色高亮）|
| Sheet: 状态变更 | 基准中存在但关键字段有变化的项目（黄色高亮）|

**输出字段**：债券名称、交易所、计划管理人、品种、拟发行金额(亿元)、项目状态、更新日期、受理日期、投放和跟进情况、备注、变更类型、变更详情

---

## 核心设计决策

### 停止条件：重复计数而非日期阈值
爬取时实时与基准索引比对，同一交易所重复项目达到 3 个即停止翻页。
*原因*：基准文件覆盖历史全量，一旦连续命中说明已进入重叠区域，继续翻页无增量价值。

### 上交所：不依赖搜索框
选择"资产支持证券(ABS)"品种后翻页，本地用关键词（"持有型"/"机构间"）过滤。
*原因*：上交所搜索框由 Vue/React 驱动，`dispatchEvent` 无法稳定触发筛选结果。

### 深交所：三级降级
优先 Excel 文件下载（最快、最稳定），失败走 API 分页，最后用 Playwright 浏览器兜底。
*原因*：交易所网站反爬策略多变，单一链路脆弱，降级保证可用性。

### 合并策略：以基准为底
不清空重建，而是在基准全量数据上做增量 patch：
- 爬取到 + 基准有 → 更新关键字段，标注变更类型
- 爬取到 + 基准无 → 追加，标"新增"
- 未爬取到 + 基准有 → 保留原样，标"无变化"
*原因*：保留人工填写的"投放和跟进情况"、"备注"等非爬取字段。

---

## 最佳实践

- 每次运行前确认 `data/input/` 中已有最新基准文件
- 首次运行建议有头模式（不加 `--headless`），便于观察浏览器行为
- 上交所网络慢时增大等待超时；深交所 Excel 链路失败属正常，会自动降级

---

## 避坑指南

| 坑 | 现象 | 解决方案 |
|---|---|---|
| 上交所搜索无效 | 关键词输入后列表不刷新 | 已放弃搜索，改为品种筛选+本地过滤 |
| 深交所 API 返回空 | `total=0` 或空 JSON | 自动降级到 Playwright 页面 |
| 基准文件字段名不一致 | `KeyError` 或比对全为新增 | `baseline_reader` 做了字段标准化，确保"交易所"列值统一为"上交所"/"深交所" |
| Excel 临时锁定文件 | `~$xxx.xlsx` 被误读 | `find_latest_baseline` 已过滤 `~$` 前缀文件 |
| `nan`/`NaT` 误判为变更 | 空字段被标注为状态变更 | `_detect_field_changes` 中已做 nan 归一化处理 |
